# Issue Rules

## 正式问题
正式问题要满足：
- 差异明确稳定
- 不太可能由截图条件造成
- 可以直接提给开发修改

常见正式问题：
- 模块类型不一致
- 核心信息结构不一致
- 对齐关系明显错误
- 关键按钮大小、颜色或位置明显不对
- 核心区域被遮挡、压叠
- 模块顺序明显错误

## 疑似问题
疑似问题用于记录：
- 看起来有偏差
- 对设计质量重要
- 但暂时不适合直接作为“必须修改”的正式问题

常见疑似问题：
- 波浪曲线疑似更硬
- 按钮光效疑似过重
- 背景层次疑似更弱
- 圆角气质疑似不同
- 局部留白节奏疑似不同

## 问题描述怎么写

### 正式问题
只写：
- 问题
- 建议

示例：
问题：开发页收益卡为双信息结构，设计稿为单核心收益结构。  
建议：按设计稿恢复收益卡信息结构与右上内容形式。

### 疑似问题
只写：
- 问题
- 建议

示例：
问题：开发页按钮外发光可能重于设计稿。  
建议：建议对该区域做局部放大比对或叠图确认。

## 不要这样写
- 写成长段小作文
- 在问题描述里重复写位置
- 在问题描述里重复写优先级和状态
- 在问题描述里塞很多背景解释
