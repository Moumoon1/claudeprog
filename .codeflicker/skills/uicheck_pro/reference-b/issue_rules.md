# Issue Rules（B端页面）

## 正式问题
正式问题要满足：
- 差异明确稳定
- 不太可能由截图条件造成
- 可以直接提给开发修改

常见正式问题类型（B端）：
- 表格列缺失或列顺序错误
- 表头标签名称不一致
- 筛选栏 / 搜索框缺失或结构不对
- 分页器样式、位置或格式不一致
- 操作按钮组缺失、顺序错误或样式不对
- Tab / 标签页切换项不一致
- 表单字段缺失或校验提示缺失
- 弹窗 / 下拉内容结构不一致
- 模块层级 / 顺序明显错误
- 关键状态标签（如状态 badge、标签颜色）不一致
- 表格行高 / 列宽明显偏差
- 切图/裁剪错误（卡片或图片边缘被截断、元素溢出被裁掉）

## 疑似问题
疑似问题用于记录：
- 看起来有偏差
- 对 B端信息密度和操作效率重要
- 但暂时不适合直接作为"必须修改"的正式问题

常见疑似问题类型（B端）：
- 表格行高疑似偏大或偏小
- 筛选区高度或间距疑似不一致
- 按钮间距疑似偏窄或偏宽
- 表头背景色或边框线疑似有偏差
- 字段标签对齐方式疑似不同
- 分割线 / 分组线疑似缺失或多余
- 空态 / 加载态展示疑似缺失

## 问题描述怎么写

### 正式问题
只写：
- 问题
- 建议

要求：
- 使用当前截图中可见、可复核的描述
- 描述差异本身，不复述 reference 里的现成业务对象
- 句子要能直接指导修改

### 疑似问题
只写：
- 问题
- 建议

要求：
- 明确这是"疑似"而非已确认事实
- 建议里写清如何进一步确认
- 仍然只能基于截图可见证据，不得凭示例补内容

## 不要这样写
- 写成长段小作文
- 在问题描述里重复写位置
- 在问题描述里重复写优先级和状态
- 在问题描述里塞很多背景解释
- 直接复制示例句子作为本次结论
