## 概述

读取用户的 PRD（产品需求文档），根据需求内容构建移动端 H5 页面。配色由用户直接指定（如主色值），或根据 PRD 内容自行选择合适的颜色。布局、组件、字体、图标等全部遵循下方强制规范。

---

## 一、配色方案

用户可提供主色值（如 `#3B82F6`），或要求自动生成。确定主色后，基于该颜色派生出完整的 UI 色阶：背景色、卡片色、内容区底色、按钮色、文字色等。深浅模式根据用户指定或内容调性自动选择。

颜色约束：
- 单色系配色，禁止多色系混用
- 按钮文字：自适应黑白（背景偏亮用黑字，背景偏暗用白字）
- 正文/标题对比度 ≥ 4.5:1（WCAG AA）
- 按钮文字对比度 ≥ 7:1（WCAG AAA）

---

## 二、布局系统（强制执行）

- 页面宽度：414px
- 最外层容器 padding：16px
- 全局禁止分割线、边框线

### 卡片通用样式
- 宽度：100%
- 圆角：16px
- 背景：card 色
- 外边距：0 0 16px 0
- 内边距：0

### 卡片内容区域
- 宽度：`calc(100% - 32px)`
- margin：16px
- 背景：card-content 色
- 内边距：16px
- 圆角：12px
- 绝对不允许贴底、不允许撑满

### 列表项
- 间距：24px
- 无分割线
- 左右布局：70% 内容 + 30% 按钮


---

## 三、字体规范

| 用途 | 字号 | 字体 | 备注 |
|------|------|------|------|
| 卡片大标题 | 20px | hyyakuhei (85J) | |
| 卡片标题区 | 22px | Alimama ShuHeiTi Bold | 也可用 hyyakuhei |
| 正文 | 14px | pingfang sc | |
| 副文案 | 12px | pingfang sc | 60% opacity |
| 标签 | 9px / 11px | pingfang sc | 小9大11 |

---

## 四、按钮规范

- 尺寸：72×36px
- 圆角：64px
- 背景：primary 色
- 文字：自适应黑白（L ≥ 70 → #000，L < 70 → #FFF）
- 字号：14px

---

## 五、图标规范

- 强制使用**内联纯 SVG 矢量图标**
- 绝对禁止任何 emoji / 表情符号
- 图标容器：48×48px 圆形容器
- 图标本体：28×28px
- 图标颜色：primary 主色，纯色无其他颜色

---

## 六、根据 PRD 构建页面的流程

1. **读取 PRD**，理解页面目标、功能模块、数据结构
2. **确定配色方案**（用户提供主色值，或根据内容自行选择）
3. **提取页面模块**：从 PRD 中识别需要哪些卡片/区域，按逻辑顺序排列
4. **填充数据**：用合理的模拟数据填充每个模块（不要用 lorem ipsum，用真实感数据）
5. **生成 HTML**：输出完整 HTML 文件，包含内联 CSS
6. **自检**：
   - 内容区是否有 16px 内外边距？
   - 是否有任何分割线/边框？（必须无）
   - 图标是否为纯 SVG？（不能有 emoji）
   - 按钮文字对比度是否 ≥ 7:1？
   - 卡片内容区是否未贴底？
   - 文字对比度是否 ≥ 4.5:1？

---

## 七、禁止行为

- 多色系配色
- 分割线、边框线
- 按钮使用固定文字颜色（必须自适应）
- 内容区缺少 16px 间距
- 内容区贴卡片底部
- 使用 emoji/表情作为图标（仅限纯 SVG）
- 使用外部 CSS/JS 文件（全部内联）
- 使用占位符文案如 "Lorem Ipsum"、"标题文字"（必须填充真实感数据）

---

## 八、输出

输出完整 HTML 文件，包含：
- 内联 CSS（所有样式写在 `<style>` 中）
- 页面宽度 414px，移动端适配 `<meta name="viewport">`
- 完整的 DOM 结构，可直接在浏览器中打开预览
