# Review Scope

这个 skill 适合做“第一轮到第二轮之间”的视觉还原走查，目标是：

- 快速找出明显没还原的部分
- 记录重要但暂时不能完全确认的设计细节
- 产出适合设计、前端、产品共同查看的 Word 问题表

## 推荐使用场景
- 移动端长截图对长图
- B 端页面开发页 vs 设计稿
- 首屏重点模块走查
- 需要输出问题表格文档
- 需要双图对照框选

## 不适合的场景
- 像素级验收
- 只差 1~2px 的对齐判断
- 字体渲染级差异验收
- 纯程序化标尺核对

## 默认重点
优先检查：
1. 页面骨架
2. 模块顺序
3. 视觉重点
4. 关键卡片 / 按钮 / 关键区块
5. 是否有明显遮挡、压叠、裁切问题
6. 重要细节疑点（波浪线、光效、背景层次等）
